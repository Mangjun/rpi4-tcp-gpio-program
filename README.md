# TCP 기반 원격 장치 제어 시스템 (Remote Device Control System via TCP)

## 1. 프로젝트 개요 (Project Overview)

### 1.1 프로젝트 소개
본 프로젝트는 TCP/IP 소켓 통신을 기반으로 원격지에 있는 임베디드 장치(라즈베리파이 4)에 연결된 다양한 하드웨어 모듈을 제어하고 모니터링하는 시스템입니다. 시스템은 다중 클라이언트 접속을 효율적으로 처리하기 위해 **멀티스레드** 기반으로 설계되었으며, 하드웨어 제어 로직을 독립적인 **동적 라이브러리**(Dynamic Library) 형태로 구성하여 유지보수성과 확장성을 극대화하였습니다. 또한 텍스트 기반 리눅스 클라이언트뿐만 아니라 사용자 편의성을 고려한 웹 클라이언트(Web Client) 인터페이스를 추가로 제공합니다.

### 1.2 시스템 구성 및 개발 환경
* **서버 (Server)**
    * **하드웨어:** `Raspberry Pi 4`
    * **운영체제:** `Linux` (Kernel: 6.18.29+rpt-rpi-v8)
    * **실행 형태:** 시스템 백그라운드에서 상시 동작하는 **데몬 프로세스**(Daemon Process)
* **리눅스 클라이언트 (Linux Client)**
    * **운영체제:** `Ubuntu 26.04` (Kernel: 7.0.0-15-generic)
    * **특징:** 프로그램 실행 중 예기치 않은 종료를 방지하기 위해 `SIGINT`를 제외한 모든 시그널을 블록킹(Blocking) 처리
* **웹 클라이언트 (Web Client)**
    * **특징:** 사용자가 웹 브라우저를 통해 편리하게 장치를 제어할 수 있도록 추가 구현된 인터페이스
* **제어 대상 장치 (Target Devices)**
    * **LED:** 원격 `On`/`Off` 제어 및 3단계 밝기 조절(최대, 중간, 최저)
    * **부저 (Buzzer):** 경고음 및 음악 소리 `On`/`Off` 제어
    * **조도 센서 (Illumination Sensor):** 주변 밝기 실시간 값 확인 및 조도에 따른 LED 자동 제어 연동
    * **7 세그먼트 (7-Segment):** 타이머 숫자(`0~9`) 표시 및 카운트다운 기능, `0` 도달 시 부저 연동
* **빌드 시스템:** `Makefile`을 통한 빌드 자동화 구성
